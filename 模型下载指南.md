## 🎯 姿态检测模型（根据你的选择下载）

项目支持3种姿态检测方案，**选择其中一种**即可：

### 方案A: YOLO Pose（推荐，最简单）⭐⭐⭐

**优点**: 
- 一体化模型，速度快
- 自动下载，无需手动配置
- GPU友好

**需要下载**: 
- `yolo11n-pose.pt` (可选下载)

**说明**: 
- 如果不下载，Ultralytics会在首次运行时**自动下载**
- 手动下载可以节省首次运行时间

**运行命令**:
```bash
python main.py \
  --video-path videos/demo.mp4 \
  --pose-family yolo-pose \
  --yolo-pose-model yolo11n-pose.pt
```

---

### 方案B: RTMPose（两阶段检测）

**优点**: 
- 检测精度可能更高
- 有3个档位可选

**需要的ONNX文件**:

#### 人体检测器（所有档位共用）:
- `yolox_nano_8xb8-300e_humanart-40f6f0d0.onnx`

#### 姿态估计器（根据档位选择一个）:
- **lightweight**: `rtmpose-t_simcc-body7_pt-body7_420e-256x192-026a1439_20230504.onnx`
- **balanced** (默认): `rtmpose-s_simcc-body7_pt-body7_420e-256x192-acd4a1ef_20230504.onnx`
- **performance**: `rtmpose-m_simcc-body7_pt-body7_420e-256x192-e48f03d0_20230504.onnx`

**说明**: 
- 如果不下载，会**自动在线下载**到缓存目录
- 手动下载需要放到 `weights/` 目录

**运行命令**:
```bash
# balanced档位（默认）
python main.py \
  --video-path videos/demo.mp4 \
  --pose-family rtmpose \
  --pose-mode balanced
```

---

### 方案C: RTMO（一阶段检测）

**优点**: 
- 一体化模型，比RTMPose更快

**需要的ONNX文件**（根据档位选择一个）:
- **lightweight**: `rtmo-s_8xb32-600e_body7-640x640-dac2bf74_20231211.onnx`
- **balanced**: `rtmo-m_16xb16-600e_body7-640x640-39e78cc4_20231211.onnx`
- **performance**: `rtmo-l_16xb16-600e_body7-640x640-b37118ce_20231211.onnx`

**说明**: 
- 如果不下载，会**自动在线下载**
- 手动下载需要放到 `weights/` 目录

**运行命令**:
```bash
python main.py \
  --video-path videos/demo.mp4 \
  --pose-family rtmo \
  --pose-mode balanced
```

## 💡 关于自动下载

代码会在这些情况下自动下载模型：

1. **YOLO Pose**: Ultralytics会自动下载到 `~/.cache/ultralytics/`
2. **RTMPose/RTMO**: rtmlib会自动下载到 `~/.cache/`

**优点**: 无需手动配置  
**缺点**: 首次运行需要联网，下载较慢

**如果自动下载失败**，再手动下载ONNX文件到 `weights/` 目录。
