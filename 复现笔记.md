# 复现笔记（WSL2 + RTX 4070 Laptop GPU）

本文记录在 **Windows WSL2 (Ubuntu) + NVIDIA RTX 4070 Laptop GPU (8GB)** 环境下从零复现本项目的完整过程，以及踩到的 4 个坑和对应解决方法。

## 环境信息

| 项 | 值 |
|---|---|
| 系统 | Windows + WSL2 (Ubuntu) |
| GPU | NVIDIA RTX 4070 Laptop GPU，8GB 显存 |
| CUDA | 12.7（Windows 驱动自带，WSL 自动可用） |
| Python | 3.8.10 |
| 最终装的版本 | torch 2.4.1+cu121 / onnxruntime-gpu 1.19.2 |

结论：**可以正常复现**，GPU 加速生效（每帧 pose+检测约 0.02s）。

## 一键分析自己的视频（推荐）

不想手动截模板、记一长串参数，用 `analyze.sh`：

```bash
# 1. 把视频放进 videos/（资源管理器拖拽或 cp 都行）
# 2. 一条命令搞定：自动生成球场模板 + 跑完整分析
./analyze.sh videos/你的视频.mp4
```

脚本会自动完成：
1. 从视频多个时间点抽帧
2. 用球场检测函数筛掉回放/特写等非比赛画面
3. 选一帧验证模板匹配分（≥0.75）后存到 `templates/<视频名>.png`
4. 配对跑完整分析，产物输出到 `outputs/<视频名>/`

**已有模板时**会自动复用 `templates/<视频名>.png`；也可手动指定：

```bash
./analyze.sh videos/你的视频.mp4 templates/自定义模板.png
```

如果自动找不到球场（机位非标准/画质问题），脚本会提示你手动截一帧标准比赛画面存到 `templates/` 再重试。单独生成模板可直接用 `python make_template.py videos/你的视频.mp4`。

| 文件 | 说明 |

## 复现过程中遇到的问题与解决

<!-- PROBLEMS_PLACEHOLDER -->

### 问题 1：requirements.txt 的版本在 Python 3.8 下装不上

**现象**：`pip install` 报错，找不到 `torch==2.5.1+cpu`、`onnxruntime-gpu==1.20.1`。

**原因**：
- PyTorch 索引里根本没有 `2.5.1+cpu` 这个版本号。
- `onnxruntime-gpu 1.20.1` 需要 Python 3.9+，而本机是 3.8。

**解决**：改用 Python 3.8 兼容、且带 CUDA 的版本：
```bash
pip install torch==2.4.1+cu121 torchvision==0.19.1+cu121 --index-url https://download.pytorch.org/whl/cu121
pip install onnxruntime-gpu==1.19.2
```
已写进 `setup.sh`，并在脚本里加了注释说明原因。

---

### 问题 2：模板用错，整段视频被当成"非球场"全部跳过（最隐蔽）

**现象**：程序显示"处理完成"，但 `detect_demo.mp4` 只有 258 字节、`detections.jsonl` 是 0 行 —— 等于什么都没处理，且不报错。

**原因**：`main.py` 中 `--template-path` 默认值是 `templates/demo1.png`，而跑的视频是 `demo.mp4`。程序靠模板匹配判断"这一帧是不是球场画面"（`is_court_view`，阈值 0.75），模板和视频对不上：
- `templates/demo.png` 对 demo.mp4 首帧匹配分 **0.969**（通过）
- `templates/demo1.png` 对 demo.mp4 首帧匹配分 **0.733**（< 0.75，每帧都被判为非球场而跳过）

**解决**：运行时务必让 `--video-path` 和 `--template-path` 配对。跑自己的视频时，先从该视频截一张球场清晰的帧作模板。

---

### 问题 3：输出视频 VSCode/浏览器打不开

**现象**：原始 `demo.mp4` 能在 VSCode 里预览，但输出的 `detect_demo.mp4` 打不开。

**原因**：OpenCV 的 `VideoWriter` 用 `mp4v`（即 mpeg4）编码写视频，而 VSCode、浏览器等内置播放器普遍只支持 `h264`。最终合成时 ffmpeg 又用了 `-c:v copy`（直接复制流、不重编码），所以输出始终是 mpeg4。

**解决**：改 `badminton_analysis/media/video_audio.py`，最终输出改用 ffmpeg 转 `libx264` + `yuv420p`（有音频 / 无音频两条路径都改），并加 `faststart` 便于网页播放。转码失败时回退为直接复制，保证至少有产物。

---

### 问题 4：分析视频里的中文统计文字全是问号

**现象**：视频左上角球员统计面板的中文显示为问号，轨迹点等图形正常。

**原因**：`badminton_analysis/visualization/stats.py` 的 `_find_chinese_font()` 只在 `C:/Windows/Fonts/` 下找字体（Windows 路径），WSL 里这些路径不存在 → `font_path` 为 `None` → 中文回退到 OpenCV 的 `cv2.putText`，而它不支持中文 → 渲染成问号。

**解决**：把项目自带的 `simhei.ttf`（仓库根目录已有）和 Linux/WSL 常见中文字体（文泉驿、Noto CJK 等）加进字体搜索列表，并把项目字体放在最前面优先使用。

## WSL 相关说明

- **图形界面可用**：本机有 WSLg（`DISPLAY=:0`、`/mnt/wslg` 存在），所以球场标注窗口、`--display true` 实时预览窗都能正常弹出。
- 若在没有 WSLg 的环境，球场标注那步会弹窗失败，可改为预置 `court_annotations.txt` 跳过交互。
- 从 Windows 拷文件进 WSL：直接在资源管理器里把文件拖进 `\\wsl$\Ubuntu\home\alon\Learning\Good-Badminton\videos`，或命令行 `cp /mnt/c/Users/<用户名>/Downloads/xxx videos/`。
- 查看输出：Windows 资源管理器地址栏输入 `\\wsl$\Ubuntu\home\alon\Learning\Good-Badminton\outputs`。
